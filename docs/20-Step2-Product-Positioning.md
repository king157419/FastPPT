# 步骤 2：产品定位（已定版）

## 1. 三档模式（与你的定义一致）

1. 代劳模式（Mode A）
   - 老师思路清晰，系统负责高质量执行与排版
2. 共创模式（Mode B）
   - 老师有大致方向，系统通过选项和建议共创
3. 全自动模式（Mode C）
   - 老师只给主题，AI 先出大纲后出课件，老师审核

## 2. V1 的战略优先级

1. 优先 Mode A（最高价值，最刚需）
2. 同步支持 Mode B（提高覆盖面）
3. Mode C 作为补充能力，不作为质量基准

核心原因：教学场景里“可控、高质量、少返工”优先于“全自动炫技”。

## 3. 产品定位一句话

FastPPT 不是“随便生成 PPT”的工具，而是“面向高校备课、强调可控与资料复用的 AI 课件协作系统”。

## 4. 差异化（要持续守住）

1. 多源资料复用：旧 PPT + 教材文档 + 图片/视频线索
2. 需求精确化引导：把模糊输入转为结构化教学规格
3. 可插拔内容块：公式块、代码块、流程图块等可独立生成/替换
4. 可追溯输出：案例和知识点尽量绑定来源
5. 课程长期记忆：按课程保存教材/案例/风格/修订记录，提高后续备课一致性

## 5. 与赛题对齐

- 多模态：文件、图片、语音、视频线索（按阶段扩展）
- RAG：资料检索与引用
- 生成：PPT + 教案
- 创新点：你们强调的“知识库/素材库可调用能力 + 可控生成流程”

## 6. 步骤 2 完成度评估

- 方向清晰度：95%
- 团队共识度：90%
- 可执行性：90%

剩余工作：把“模式选择 -> 参数化配置 -> 生成行为差异”落成接口与前端交互。

补充定位约束（2026-04-22）：

1. 生成前的必确认项固定为：教学目标 / 学生类型 / 重点难点
2. 结果验收以“内容正确、结构顺、像老师自己的课、可回溯来源”为硬标准
