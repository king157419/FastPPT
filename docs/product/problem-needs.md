## 1. 真实场景起点（已确认）

高校老师最常见输入起点：

1. A：网上能找到同课程 PPT，需要更新改造（最常见）
2. B：自建新课，网上没有可直接复用课件（次常见）

结论：V1 必须先把 A 场景做到“省心且可控”，同时覆盖 B 的从 0 到 1。

补充（已确认）：

- 在已有 PPT 场景里，最高频动作是“替换过时内容 + 加新案例/新政策 + 插图表/视频片段”
- “多资料合并成新课件”也常见，但难度更高，应作为增强能力设计

## 2. 痛点（按影响排序）

1. 需求表达模糊，AI 输出不可控，返工成本高
2. 旧课件与教材资料利用率低，更新“最新案例”费时
3. 对“花哨但不相关”的输出容忍度极低
4. 图文质量、相关性和可信度难兼顾（担心幻觉）
5. 多种素材（文字/图片/公式/代码/流程）插入成本高

## 3. 老师明确偏好（可直接转产品要求）

1. 先引导再生成：支持勾选式输入，减少模糊 prompt
2. 内容要图文并茂，但不追求花哨动画
3. 希望可插入“示意图 + 最新应用案例”
4. 资料引用要可追溯，避免幻觉
5. 可以接受后续修改动作（修改是刚需而不是异常）
6. 最常见修改是“页级改动”，其次是“加案例”和“删复杂内容”
7. 对“课程长期记忆（教材/案例/风格/修订记录）”接受度高，且暂无显性顾虑

## 4.1 生成前必确认信息（新增硬约束）

1. 教学目标
2. 面向学生类型
3. 重点难点

## 4. 需求优先级（MoSCoW）

Must：

- 旧 PPT 更新与结构保留
- 教材/讲义/论文等资料可检索复用
- 可控生成（风格、密度、案例、图文比例可选）
- 低幻觉与引用依据

Should：

- 公式块、代码块、流程图等结构化内容插入
- 最新案例推荐与替换
- 局部重生成（按页/按块）

Could：

- 更高级互动模板
- 复杂动画与演讲者模式增强

Won’t（当前阶段）：

- 重型“花哨展示”优先级低于可控与正确

## 4.2 “可用”验收标准（新增）

1. 内容正确
2. 结构顺
3. 风格像老师自己的课
4. 能回到给定资料，不乱编
