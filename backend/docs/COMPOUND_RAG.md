# FastPPT 复利RAG系统说明

## 🎯 什么是复利RAG？

复利RAG（Compound RAG）是Andrej Karpathy提出的概念，核心思想是：
**知识像复利一样累积，每次交互都让系统变得更好**

### 传统RAG vs 复利RAG

**传统RAG**：
```
用户查询 → 检索文档 → 生成回答 → 结束
（每次都是独立的，没有学习）
```

**复利RAG**：
```
用户查询 → 检索文档 + 历史模式 → 生成回答 → 提取新模式 → 存储
                ↑                                              ↓
                └──────────────── 知识累积 ─────────────────────┘
（每次交互都让系统更好，知识产生复利）
```

## 📊 FastPPT的实现

### 1. 知识存储结构

```
.fastppt/knowledge/
├── index.json              # 知识索引
├── entries/                # 知识条目
│   ├── 光合作用_初中.md
│   ├── 牛顿定律_高中.md
│   └── ...
```

### 2. 复利效应示例

**第1次生成"光合作用"课件**：
- 系统：从零开始生成
- 质量：6/10
- 存储：提取教学模式（如何讲解光合作用）

**第2次生成"光合作用"课件**：
- 系统：使用上次的教学模式
- 质量：7/10
- 存储：精炼模式（合并新旧知识）

**第10次生成"光合作用"课件**：
- 系统：使用10次精炼的模式
- 质量：9/10
- 复利因子：10（平均每个主题更新10次）

### 3. 跨课程学习

```
生成"光合作用"课件 → 学到"生物实验讲解模式"
                    ↓
生成"化学实验"课件 → 复用"实验讲解模式"
                    ↓
                  质量提升！
```

## 🔧 API使用

### 查看复利统计

```bash
curl http://localhost:8000/api/knowledge/stats
```

返回：
```json
{
  "total_entries": 50,
  "total_updates": 200,
  "compound_factor": 4.0,  // 平均每个主题精炼4次
  "total_tags": 20,
  "total_connections": 100
}
```

### 搜索知识

```bash
curl -X POST http://localhost:8000/api/knowledge/search \
  -H "Content-Type: application/json" \
  -d '{"query": "光合作用", "top_k": 5}'
```

### 生成课件（自动学习）

```bash
curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "intent": {
      "topic": "光合作用",
      "audience": "初中生"
    }
  }'
```

返回包含：
```json
{
  "slides_json": {...},
  "compound_factor": 4.2,  // 系统已学习4.2倍
  "message": "课件生成成功"
}
```

## 💡 关键特性

1. **自动学习**：每次生成后自动提取教学模式
2. **知识精炼**：重复主题会合并和改进知识
3. **跨课程迁移**：一个科目的经验帮助其他科目
4. **复利追踪**：实时显示系统改进程度
5. **持久化**：知识永久保存，重启不丢失

## 🎓 教学场景示例

**场景1：新老师使用**
- 第1次：系统从零开始，质量一般
- 第10次：系统学会了你的教学风格
- 第50次：系统成为你的"教学助手"

**场景2：多老师共享**
- 老师A：擅长实验讲解
- 老师B：擅长理论推导
- 系统：学习两者优点，综合最佳实践

**场景3：跨学期改进**
- 上学期：生成100份课件
- 本学期：质量自动提升30%
- 原因：系统已经学习了100次
