# Capability: course-memory

## What it is

The planned course-scoped memory layer for storing recurring materials, style preferences, cases, and revision history.

## Read when

- planning differentiation work
- designing persistence beyond single-generation sessions
- deciding what long-term teacher value should look like

## Source of truth

- `docs/product/product-positioning.md`
- `docs/engineering/feature-decomposition-and-implementation.md`

## Current status

- `planned`

## Intended behavior

- Reuse course materials across sessions
- Learn recurring style and structure preferences
- Retain useful revision history with explicit controls

## Risks

- Memory without clear boundaries will quickly become noisy.
- It needs auditability and opt-out controls to be acceptable in education workflows.
