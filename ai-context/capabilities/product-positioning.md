# Capability: product-positioning

## What it is

The stable definition of what FastPPT is, which user problems it solves first, and which modes matter in V1.

## Read when

- deciding feature priority
- evaluating whether a change helps or distracts
- aligning demo language with product truth

## Source of truth

- `docs/product/product-positioning.md`

## Current status

- `active`
- Product story is clear
- Engineering still has gaps versus the ideal Mode A experience

## Key conclusions

- FastPPT is an AI courseware collaboration system, not a generic PPT generator.
- V1 should prioritize Mode A and Mode B.
- Mode C is useful, but it is not the benchmark for quality.
- Long-term differentiators are source reuse, structured clarification, block-based content, evidence, and course memory.
